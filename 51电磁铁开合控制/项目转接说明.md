# 51 单片机三路电磁铁控制项目转接说明

> 用途：供新的 Codex 对话快速接手项目。请先完整阅读本文，再检查实物、原理图和开发板资料，不要直接套用旧 STM32 程序。
>
> 新项目修改边界：所有新增、修改和构建输出仅放在 `D:\物流比赛\51电磁铁开合控制`，不要修改旧的 `D:\物流比赛\电磁铁开合控制` 及其他项目。

## 1. 新项目目标

使用一块 51 单片机开发板控制四路光耦 MOS 模块中的前三路，从而分别控制三只 12V 电磁铁：

- 通电：电磁铁吸合。
- 断电：电磁铁释放。
- 上电、复位、下载程序时默认三路全部断开。
- 通过串口命令独立控制三路。
- 第一版不做 PWM 调力、上位机界面和自动流程。

计划保留的串口命令：

```text
M1 ON
M1 OFF
M2 ON
M2 OFF
M3 ON
M3 OFF
ALL ON
ALL OFF
STATUS
HELP
```

建议串口参数：`9600、8N1`，命令以 CRLF 结束。新板使用 11.0592MHz 晶振，9600 波特率实现简单且稳定。

### 2.2 MOS 驱动模块

模块为四路光耦 MOS 控制板，使用前三路：

控制侧端子：

```text
PWM1 / GND1
PWM2 / GND2
PWM3 / GND3
PWM4 / GND4（不用）
```

功率侧端子：

```text
DC+ / DC-
OUT1+ / OUT1-
OUT2+ / OUT2-
OUT3+ / OUT3-
OUT4+ / OUT4-（不用）
```

旧资料确认 MOS 控制输入建议至少约 3.6V，不能依赖 3.3V GPIO 直接高电平驱动。51 板运行在 5V，更适合继续采用“5V 正端固定、GPIO 负责下拉”的低电平有效方案。

MOS 资料仍在旧目录，仅供参考，不应修改：

```text
D:\物流比赛\电磁铁开合控制\mos控制板\mos控制板说明书.jpg
```

### 2.3 电磁铁和电源

- 电磁铁额定电压：12V。
- 三只电磁铁分别接 OUT1、OUT2、OUT3。
- 电磁铁线圈本身不强调正负，但应按 MOS 板 OUT+、OUT- 成对连接，避免绕过板载续流保护。
- MOS 功率输入：12V 正极接 DC+，12V 负极接 DC-。
- 12V 电源容量需满足三路同时吸合，并预留约 30% 余量。
- 51 开发板和 MOS 控制侧使用 5V，电磁铁功率侧使用 12V，两者作用不同。

### 2.4 USB-TTL

另有一块外置 USB-TTL：

- 型号标签：`DSTUT-CH340G6AMN`。
- 引脚：`GND、RXD、TXD、CTS、RTS、VCC`。
- 带 3.3V/5V VCCIO 电平选择。
- 驱动：`D:\物流比赛\电磁铁开合控制\CH341SER.EXE`。
- 当前电脑已正常识别为 `USB-SERIAL CH340 (COM5)`；COM 号以后可能变化，应以设备管理器为准。

由于 SmartMetal 51 板原理图已经包含板载 CH340G，外置 USB-TTL 很可能不需要。接手时优先验证板载 USB 下载；只有板载下载不可用时，才考虑外置 USB-TTL。

如果使用外置 USB-TTL 连接 5V STC89C52RC：

```text
USB-TTL TXD -> STC P3.0/RXD
USB-TTL RXD -> STC P3.1/TXD
USB-TTL GND -> 51 板 GND
USB-TTL VCC -> 不接（51 板自己供电）
```

串口信号电平选择 5V。TX、RX 必须交叉连接。

## 3. 推荐的新硬件方案（待结合实物最终确认）

建议先考虑以下三路 GPIO：

```text
P1.0 -> MOS GND1
P1.1 -> MOS GND2
P1.2 -> MOS GND3
```

MOS 输入正端连接：

```text
51 板 5V -> PWM1
51 板 5V -> PWM2
51 板 5V -> PWM3
```

控制逻辑：

| 逻辑状态 | P1.x 状态 | MOS 输入 | 电磁铁 |
| --- | --- | --- | --- |
| OFF | 写 1 / 释放 | 无有效输入电流 | 断开 |
| ON | 写 0 / 下拉 | 约 5V 输入电压 | 吸合 |

选择 P1 口而不是 P0 口，是因为经典 8051 的 P0 为开漏结构，作为普通 I/O 通常需要外部上拉。P1 口更适合第一版验证。

重要：上述 P1.0、P1.1、P1.2 是候选方案。开始接线前必须结合 SmartMetal 实物确认这些排针没有被已安装的 LCD、传感器或其他模块占用。原理图中部分显示模块会使用 P1.0/P1.1，但如果没有插接显示模块，一般不会形成冲突。

功率侧保持：

```text
12V 正极 -> DC+
12V 负极 -> DC-

电磁铁 1 -> OUT1+ / OUT1-
电磁铁 2 -> OUT2+ / OUT2-
电磁铁 3 -> OUT3+ / OUT3-
```

控制侧与功率侧通过光耦隔离。第一版不要自行把 12V 负极连接到 51 板 GND，除非 MOS 模块资料明确要求。

## 4. 新程序的实现要求

当前 STM32 的 HEX 和源码不能用于 89C52。新工程需重新建立为 Keil C51 工程，目标芯片选 `STC89C52RC` 或兼容的标准 `Intel 8052`，头文件通常使用 `reg52.h`。

建议模块划分：

```text
main.c              主循环和命令分发
magnet.c/.h         三路电磁铁状态与低电平有效封装
uart.c/.h           串口初始化、接收缓冲和发送
```

程序必须满足：

1. 上电第一时间将 P1.0、P1.1、P1.2 写 1，三路默认 OFF。
2. 程序逻辑状态与硬件低电平有效分离，业务层不直接散落写端口。
3. 串口支持前述命令，不区分大小写可作为后续增强；第一版至少正确识别大写命令。
4. 错误命令只返回错误，不改变当前输出。
5. `STATUS` 返回三路逻辑状态。
6. 接收缓冲要有长度上限，避免越界。
7. 先使用 9600、8N1 和 CRLF；稳定后才考虑更高波特率。
8. 生成新的 89C52 专用 HEX，不能烧录旧的 `MagnetControl.hex`。

## 6. 已完成的实验过程与结果

### 6.1 MOS 与电源认识

已确认：

- MOS 模块只是电子开关，不会把 5V 升压成 12V。
- 5V负责单片机和MOS输入控制。
- 12V负责电磁铁实际吸合功率。
- 不接12V时，MOS控制侧指示灯仍可能亮，因为指示灯由控制侧5V驱动；这不代表电磁铁功率侧已经通电。
- 12V未接时，电磁铁不会吸合。

### 6.2 USB-TTL 驱动和端口

已完成：

- 安装/识别 CH340 驱动。
- Windows 设备管理器曾显示 `USB-SERIAL CH340 (COM5)`。
- 外置 CH340 插电脑后指示灯亮。
- FlyMcu 能打开 COM5，但 STM32 没有回复。

经验：USB-TTL灯亮只说明获得USB供电；设备管理器出现COM口只说明电脑识别转换器，不代表目标单片机已经正常通信。

### 6.3 STM32 下载尝试

曾使用 FlyMcu：

- COM5。
- 115200。
- 选择旧的 `MagnetControl.hex`。
- 取消了“编程到FLASH时写选项字节”，避免意外修改芯片选项字节。
- STM32下载需要 BT0/BOOT0=1、BT1/BOOT1=0，然后复位进入系统BootLoader。
- USB-TTL接法确认是 TXD->PA10、RXD->PA9、GND->GND，VCC不接。

FlyMcu日志一直显示开始连接但“接收到”为空。后来确认根本原因不是程序文件，而是STM32板3.3V电源异常，芯片没有正常工作。

注意：以上 BOOT0/FlyMcu 流程是 STM32 专用经验，不要照搬到 STC89C52RC。STC 使用自己的 ISP 下载机制，通常由 STC-ISP 等工具等待单片机冷启动。

### 6.4 STM32 板故障定位

现象：

- 外部5V接入时，D1电源灯微弱闪一下后熄灭。
- 只保留5V正负两根供电线，现象仍然存在。
- 四个5V针脚对GND均稳定约5V。
- 四个3.3V针脚对GND均只有约0.7V。
- 直接从3.3V针脚外供3.3V时，D1可以亮，但这种方式会绕过并可能反向供电给故障稳压器，不适合作为正式方案。

判断：

- STM32板的3.3V电源轨异常。
- 较可能是板载 AMS1117-3.3 稳压器损坏，也不排除3.3V轨其他器件短路。
- 因此停止继续用该板烧录和驱动电磁铁，转向STC89C52RC方案。

安全经验：

- 电源指示灯闪一下熄灭属于电源塌陷迹象，不能反复通电。
- 5V输入正常不代表3.3V核心电源正常，必须分别测5V和3.3V。
- 不要从3.3V针脚强行供电作为长期修复。

## 7. 建议的新项目实施顺序

### 阶段一：确认51板和下载环境

1. 拍摄51开发板正反面和芯片丝印。
2. 确认芯片确实为 STC89C52RC。
3. 阅读 SmartMetal 原理图，确认USB接口、供电开关、自动下载电路和排针方向。
4. 插电脑后检查板载CH340对应的COM口。
5. 确认配套工具是 YahBoom 工具还是需要 STC-ISP。
6. 从现有LED例程建立最小工程，先验证编译、下载和一颗LED闪烁。

### 阶段二：串口验证

1. 不连接MOS和12V。
2. 实现9600、8N1串口收发。
3. 上电打印启动信息。
4. 用SSCOM发送 `HELP`、`STATUS`，确认CRLF和回显。

### 阶段三：MOS输入侧验证

1. 仍不接12V。
2. 连接5V到PWM1/PWM2/PWM3。
3. 连接三路候选GPIO到GND1/GND2/GND3。
4. 分别发送M1/M2/M3命令，观察MOS输入指示灯。
5. 复位后确认三路指示灯全部关闭。
6. 如果指示灯常亮，优先检查端口初始化、低电平有效逻辑和接线是否插错。

### 阶段四：12V和电磁铁

1. 完全断电后连接DC+、DC-和一只电磁铁。
2. 先上5V并发送 `ALL OFF`，再上12V。
3. 单路只吸合1到2秒，随后关闭。
4. 依次验证三路。
5. 最后短时间测试 `ALL ON`。
6. 检查12V压降、单片机是否复位、MOS和线圈温升。

## 8. 关键安全规则

- 所有改线必须在5V和12V都断电时进行。
- 12V绝不能进入单片机5V、3.3V或GPIO。
- MOS板DC+、DC-不能接反。
- USB-TTL或板载CH340只负责通信；如果开发板已自己供电，不再连接外置USB-TTL的VCC。
- 不要使用万用表电流档直接跨接电源正负极。
- 第一次验证不接12V，只看串口和MOS指示灯。
- 电磁铁是否允许长期通电必须以实际规格为准；早期测试只做短时吸合。
- 上电和复位默认OFF是必要安全要求，不是可选功能。

## 9. 与树莓派连接的后续注意事项

STC89C52RC和SmartMetal板工作在5V，而树莓派GPIO是3.3V且不能承受5V：

- STC TXD到树莓派RXD之间必须做5V到3.3V电平转换。
- 树莓派TXD到STC RXD也建议通过双向电平转换模块，避免3.3V高电平阈值不可靠。
- 两侧通信时需要共地。
- 第一阶段先使用电脑串口完成验证，树莓派集成后做。
