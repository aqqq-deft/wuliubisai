# STC89C52RC 三路电磁铁串口控制验证程序

## 1. 用途与安全结论

本工程用于让 STC89C52RC 通过四路光耦 MOS 模块的前三路，独立控制三只 12V 电磁铁通断。

- 电磁铁吸合：对应 GPIO 输出低电平。
- 电磁铁释放：对应 GPIO 输出高电平。
- 单片机上电、复位和进入程序时，三路默认全部关闭。
- 第一版只做通断控制，不做 PWM 调力、自动流程或上位机界面。

严禁把 12V 接入开发板的 DC5-9V、5V、3.3V 或任意 GPIO。所有改线必须在 5V 和 12V 都断电时进行。

## 3. GPIO 与 MOS 控制侧接线

| STC89C52RC 开发板 | MOS 模块 | 作用 |
| --- | --- | --- |
| 5V | PWM1 | 第 1 路光耦输入正端 |
| P2.0 | GND1 | 第 1 路低电平吸合 |
| 5V | PWM2 | 第 2 路光耦输入正端 |
| P2.1 | GND2 | 第 2 路低电平吸合 |
| 5V | PWM3 | 第 3 路光耦输入正端 |
| P3.3 | GND3 | 第 3 路低电平吸合 |

注意：

1. `GND1/GND2/GND3` 是三个光耦输入的负端，只连接对应 GPIO。
2. 不要再把 `GND1/GND2/GND3` 接开发板 GND，否则对应通道会常开。
3. P2.0、P2.1没有板载负载；P3.3只连接未插装的 LCD12864 接口。
4. 如果以后插上 LCD12864 模块，必须先停用或重新分配 P3.3。

## 4. GPIO 直连前的强制测量

STC 手册给 P1/P2/P3 在 5V 下的低电平灌电流保证值约为 4mA。当前按直接连接方案设计，但必须先测 MOS 光耦输入电流。

逐路执行：

1. 不接 12V，不接电磁铁，不连接 GPIO。
2. MOS 的 `PWMn` 接开发板 5V。
3. 万用表切换到直流毫安档，串联在 MOS 的 `GNDn` 与开发板 GND 之间。
4. 上 5V 后读取电流，测完立即断电，再测下一路。
5. 三路都不超过 3mA，才允许按第 3 节直接连接 GPIO。
6. 程序发送 `Mn ON` 后，再测对应 GPIO 对开发板 GND 的电压，应不高于 0.4V。

只要任一路超过 3mA、低电平超过 0.4V或测量不稳定，就停止直连，改用 ULN2003A 或三只 NPN 三极管下拉。不要为了省元件让单片机端口长期超负荷工作。

## 5. 外置 USB-TTL 与 5V 供电

首次验证按“独立稳压 5V + 外置 USB-TTL”接线：

| 外置 USB-TTL | 51 开发板 |
| --- | --- |
| TXD | P3.0 / RXD |
| RXD | P3.1 / TXD |
| GND | GND |
| VCC | 不接 |

- USB-TTL 电平选择 5V。
- 稳压 5.0V 正极接开发板 `5V` 排针，负极接开发板 `GND`。
- 外部 5V 供电时，不连接开发板板载 Micro-USB，避免两路 5V 并联。
- 下载与串口调试期间，外置 USB-TTL 的 VCC 始终不接。

## 6. 12V 功率侧接线

```text
12V 电源正极 -> MOS DC+
12V 电源负极 -> MOS DC-

电磁铁 1 两根线 -> OUT1+ / OUT1-
电磁铁 2 两根线 -> OUT2+ / OUT2-
电磁铁 3 两根线 -> OUT3+ / OUT3-
```

电磁铁线圈自身不区分哪根线作为正极，但接入模块后必须按每路 `OUT+`、`OUT-` 成对连接。不要绕过模块端子，也不要把独立 12V 的 `DC-` 与开发板 GND 连接。

12V 电源额定电流至少为三只电磁铁实测工作电流之和的 1.3 倍。模块标称“每路 10A”不等于接线端子、散热和电源可以长期按 10A 使用。

## 7. 串口协议

串口设置：

```text
9600 baud
8 data bits
No parity
1 stop bit
行结束：CR、LF 或 CRLF
```

程序只识别大写命令：

```text
M1 ON
M1 OFF
M2 ON
M2 OFF
M3 ON
M3 OFF
ALL ON
ALL OFF
STATUS
HELP
```

返回示例：

```text
READY
OK
M1=ON M2=OFF M3=OFF
ERR UNKNOWN
ERR LINE TOO LONG
```

- 通断命令成功后返回 `OK`。
- `STATUS` 返回三路逻辑状态。
- 未知命令只报错，不改变输出。
- 一行最多 31 个字符；超长内容会丢弃到本行结束，不会写越界。

## 8. Keil C51 编译

具备 Keil C51 后：

1. 用 Keil uVision 打开 `MagnetControl51.uvproj`。
2. 确认 Target 为 `AT89S52`，晶振显示 `11.0592 MHz`。
3. 确认三个源文件都在 `Source Files` 组中。
4. 执行 `Project -> Rebuild all target files`。
5. 目标结果应为 `0 Error(s), 0 Warning(s)`。
6. 新 HEX 应生成到：

```text
D:\物流比赛\51电磁铁开合控制\电磁铁串口控制验证程序\Objects\MagnetControl51.hex
```

如果本机 Keil 设备库没有 AT89S52，可以选择兼容的标准 Intel 8052；不要改成 STM32 目标。

## 9. 分阶段验证

### 阶段一：只验证单片机和串口

1. 不接 MOS，不接 12V。
2. 下载程序并重新给 51 板上电。
3. SSCOM 选择正确 COM 口、9600、8N1，打开串口。
4. 应收到 `READY`。
5. 发送 `HELP`、`STATUS` 和全部开关命令，检查返回内容。

### 阶段二：只接 MOS 控制侧

1. 完成第 4 节输入电流测量并确认满足直连条件。
2. 仍不接 12V和电磁铁，按第 3 节接三路输入。
3. 上电或复位后，三路 MOS 输入指示灯应全部熄灭。
4. 依次发送 `M1 ON/OFF`、`M2 ON/OFF`、`M3 ON/OFF`，每次只能有对应通道变化。
5. 发送 `ALL OFF` 后三路应全部熄灭。

### 阶段三：逐路接入 12V 电磁铁

1. 5V 和 12V 全部断电后，只接第一只电磁铁。
2. 先上 5V，确认程序返回 `READY`，发送 `ALL OFF`。
3. 再上 12V，发送 `M1 ON`，只吸合 1到2秒后立即发送 `M1 OFF`。
4. 断电后换到第二、第三路，重复测试。
5. 三路单独正常后，才短时间执行 `ALL ON`。
6. 检查 12V 压降、开发板是否复位、MOS 与线圈温升；发现异常立即断电。
