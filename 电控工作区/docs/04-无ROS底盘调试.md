## 三、烧录

1. 电池保持断开，四轮离地。
2. 打开FlyMcu，选择设备管理器中 `USB-SERIAL CH340` 对应的COM口，波特率115200。
3. 选择上面的 `project.hex`。
4. 勾选“校验”“编程后执行”“编程前重装文件”。
5. 不勾选“使用RamIsp”“连续烧录模式”“编程到FLASH时写选项字节”。
6. 断开Type-C，按住主板中央S4/BOOT，插入Type-C并继续按约8秒，然后松开。
7. 开始编程，必须看到写入100%、校验成功和从 `0x08000000` 运行。
8. 关闭FlyMcu，拔掉Type-C等待5秒；不按S4，正常重新插入Type-C。
9. 确认程序LED持续闪烁且四轮不动。

## 四、架空动作验收

双击：

```text
D:\物流比赛\电控工作区\tools\启动底盘调试工具.cmd
```

连接CH340后，先保持100 mm/s和0.30 rad/s。每个动作首次只按约1秒：

1. 前进 -> 松开 -> 后退 -> 松开。
2. 左横移 -> 松开 -> 右横移 -> 松开。
3. 原地左转 -> 松开 -> 原地右转 -> 松开。
4. 前进左转 -> 松开 -> 前进右转 -> 松开。
5. 点击急停，确认立即停止。
6. 最后测试断连保护：保持最低速运动，停止发帧或拔掉Type-C，必须在500 ms内停车。

任一动作出现方向错误时，立即松开并记录端口和轮位；不要继续测试后续动作。出现异常发热、异味、冒烟、尖锐啸叫、轮子持续满速或急停无效时，立即断开电池。

## 五、通过后需要反馈

把以下结果发给Codex，再决定是否只改轮序、方向或PID：

- A/B/C/D分别对应哪个轮位。
- 八个动作中四个轮子的实际转向。
- 松开按钮是否立即停车。
- 断连后是否在500 ms内停车。
- 工具日志中收到的十六进制反馈数据。
- 是否有某一轮明显更慢、抖动或不转。

在这些实测结果确认前，不提高PWM上限，不落地行驶，不调整PID。
